# Contributing to awesome-shadcn-admin

Thanks for contributing. A few rules to keep the list high quality.

## What qualifies

- Built with shadcn/ui (not just Tailwind or Radix alone)
- Admin-facing UI — dashboards, CMS admin panels, SaaS back-offices, internal tools
- Open source (GitHub repo required)
- Active maintenance — at least one commit in the past 12 months
- Working demo or live site preferred

## What doesn't qualify

- Personal portfolio sites
- Tutorial repos / code samples from blog posts
- Archived or abandoned projects
- Libraries that just wrap shadcn components with no new functionality

## How to add an entry

1. Fork this repo
2. Add your entry to the correct section in `README.md`
3. Format: `- [Name](github-url) - One sentence description. ★star-count`
4. Keep descriptions factual — no marketing language
5. Sort alphabetically within each section (ignore "the", "a", "an")
6. Open a PR with title: `Add [Project Name]`

## PR checklist

- [ ] Entry is in the right section
- [ ] Description is one sentence, ends with a period
- [ ] GitHub link goes to the repo (not the website)
- [ ] Star count is current (check before submitting)
- [ ] No duplicate of an existing entry
- [ ] Alphabetical order maintained
